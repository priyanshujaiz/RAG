🧠 Project Title
OpenSource Atlas
AI-Powered Open-Source Intelligence & Contribution Platform
________________


📌 Project Overview (Problem → Solution)
❌ The Problem Today
Open-source beginners (GSOC aspirants, students, early developers) face:
* Information overload on GitHub

* No clear signal of which repositories are alive

* No guidance on where their skills fit

* No explanation of recent code changes

* High barrier to first contribution

Existing tools show raw statistics, not actionable intelligence.
________________


✅ The Solution
OpenSource Atlas is an AI-powered platform that:
   * Continuously ingests GitHub repository data

   * Computes real activity & health metrics

   * Uses AI to explain changes, trends, and risks

   * Provides a chat interface grounded in repository data

   * Recommends repositories and issues personalized to users

This transforms GitHub from data → understanding.

🚪 API Gateway Layer (Node.js)
Node.js + NestJS
Why NestJS
            * Clean architecture

            * Dependency injection

            * Scalable backend patterns

            * Used in serious production systems

Responsibilities

               * User management

               * Rate limiting

               * API aggregation

               * WebSocket support (live updates)

Why Node here

                  * Acts as gateway, not AI logic

🔄 Data Flow Example (End-to-End)
User asks:
“Which repos are best for GSOC beginners in ML?”
Flow
                                                                                                      1. Frontend sends request to Node

                                                                                                      2. Node authenticates user

                                                                                                      3. Node calls Python AI service

                                                                                                      4. Python:

                                                                                                         * Queries metrics from PostgreSQL

                                                                                                         * Retrieves embeddings from Vector DB

                                                                                                         * Agent reasons & ranks repos

                                                                                                            5. LLM generates explanation

                                                                                                            6. Response returned to frontend
